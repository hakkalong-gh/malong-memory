## 🎯 一、核心决策（已固化）

| 维度 | 决策 | 理由 |
|------|------|------|
| **定位** | AI 联合授课老师（Co-Teacher） | 学校敢试、教师能接、效果可验、后续可扩 |
| **目标** | 深中 3-5 节样板课（样板验证） | 非大而全平台，先做可演示的最小教学闭环 |
| **用户** | 教师第一，学生第二，校长第三 | 教师是关键决策影响者 |
| **场景** | 标准班级授课（先打透这一个） | 聚焦，不摊大饼 |
| **信任** | 教师一键接管 + 内容可追溯 | 教育场景控制权比自动化更重要 |
| **品牌** | 码隆 AI 授课基础设施 | 有系统感、规模感、B 端价值 |
| **MVP** | 单课闭环（非大平台） | 5-10 分钟演示看懂整个产品逻辑 |
| **学科** | 数学/物理优先 | 逻辑清晰、互动点好设计、易标准化 |

## 🎯 四、P0 MVP 拆解（10 模块）

### 第一优先级（7 个，核心闭环）

1. **课程配置模块** — 教师 3-5 分钟定义一节 AI 课
2. **教学资料接入模块** — 上传教材/PPT/题目，AI 不脱离真实内容
3. **AI 教案生成模块** — 把目标 + 资料转成结构化课堂流程（产品中枢）
4. **教师审核编辑模块** — 教师可修改/删除/锁定，建立信任
5. **课堂主讲模块** — AI 按教案节点完成主讲
6. **教师控制台模块** — 暂停/继续/重讲/跳过/接管（第一优先级）
7. **课后复盘模块** — 自动生成课堂结果与复盘建议（价值收口）

### 第二优先级（3 个，增强体验）

8. **学生互动模块** — 选择题 + 懂/不懂反馈
9. **基础即时反馈模块** — 答题统计 + 重讲/继续建议
10. **最小数据记录模块** — 支撑复盘与迭代

## 💡 八、关键洞察（开发过程记录）

1. **教师控制台是第一优先级** — 比 AI 主讲更重要，决定学校敢不敢用
2. **AI 教案生成是产品中枢** — 把教学目标 + 教材转成结构化课堂执行计划
3. **课后复盘是价值收口** — 决定校长能不能看到管理价值
4. **单课闭环 > 大平台** — 5-10 分钟演示看懂整个产品逻辑
5. **OpenMAIC 集成显著提升质量** — RAG + TTS + 多 Agent 让原型更接近真实产品

## 🎯 十一、讨论质量评估（事后复盘）

### 为什么这次讨论质量高？

| 维度 | 表现 | 原因 |
|------|------|------|
| **问题定义清晰** | DL 一开始就要求"先检索产品设计方法论" | 避免盲目讨论，先选框架 |
| **渐进式深入** | 产品定义 → 模块设计 → MVP 拆解 → 演示脚本 → 页面原型 → 开发 | 每层都基于上一层的共识 |
| **决策有依据** | 每个选择都有"自我辩论"环节（观点 A vs 观点 B） | 不是拍脑袋，是权衡后的选择 |
| **可执行性强** | 每层产出都能直接交给产品/研发执行 | 不是空谈概念，是落地方案 |
| **时间紧凑** | 约 3 小时完成从 0 到可运行原型 | 聚焦，不跑题 |
| **OpenMAIC 集成** | 及时利用已有技术资产（昨晚刚部署） | 不重复造轮子，站在巨人肩膀上 |

### 可复用的讨论模式

```
1. 选框架（方法论检索） → 2. 定方向（6 维度自我辩论） → 3. 出方案（产品定义）
→ 4. 拆模块（架构图） → 5. 定 MVP（优先级） → 6. 写脚本（演示流程）
→ 7. 画页面（原型清单） → 8. 开发现（可运行原型）
```

### 关键转折点

| 时间点 | 转折点 | 影响 |
|--------|--------|------|
| 定位讨论 | DL 说"不是 AI 替代教师，是 AI 增强教师" | 定下产品基调，避免跑偏 |
| MVP 范围 | 决定做"单课闭环"而非"大平台" | 确保 8 分钟演示能看懂 |
| 教师控制台 | 小爪判断"这是第一优先级，比 AI 主讲更重要" | 建立学校信任的关键 |
