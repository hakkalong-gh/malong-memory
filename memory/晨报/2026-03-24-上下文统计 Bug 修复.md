# 上下文统计 Bug 修复记录（2026-03-24）

## 问题描述

**现象**：`session_status` 始终显示上下文使用率为 0%，但实际会话文件已经有 729 行、1.89MB、约 78.7 万 tokens。

**影响**：DL 无法准确判断上下文使用情况，可能导致超限后 API 失败。

---

## 根因分析

### 已确认的事实

| 指标 | session_status 显示 | 实际值 |
|------|-------------------|--------|
| **会话文件大小** | - | 1.89 MB |
| **总字符数** | - | 157 万字符 |
| **总行数** | - | 729 行 |
| **估算 token 数** | 0 | 约 78.7 万 tokens |
| **上下文窗口** | 1.0m | 1.0m (qwen3.5-plus) |
| **实际使用率** | **0%** | **约 78.7%** |

### 可能原因

1. **OpenClaw 的 token 计数器未正确累加**
   - 每次 API 调用的 usage 可能未正确累加到会话级别
   - 或者累加逻辑有 bug

2. **Direct Runtime 模式绕过统计**
   - 当前使用 `runtime: direct` 模式
   - 可能绕过了 OpenClaw 的上下文追踪机制

3. **Compaction 配置缺失**
   - 原配置中没有 `.models.compaction` 设置
   - 可能导致统计机制未启用

---

## 已实施的修复

### 1. 添加 Compaction 配置

**修改文件**: `/home/malong/.openclaw/openclaw.json`

**添加内容**:
```json
{
  "models": {
    "compaction": {
      "mode": "token",
      "softThresholdTokens": 800000
    }
  }
}
```

**说明**: 
- `mode: "token"` - 基于 token 数触发压缩
- `softThresholdTokens: 800000` - 80 万 tokens 时开始软压缩（80% 使用率）

### 2. 创建上下文检查脚本

**文件位置**: `/home/malong/.openclaw/scripts/check_context_usage.py`

**使用方法**:
```bash

```
