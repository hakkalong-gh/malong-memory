## 风险识别

### 问题 1：机制叠加 → 思考包袱加重

**当前会话启动流程**：
```
1. 读取 SOUL.md
2. 读取 USER.md
3. 读取 MEMORY.md
4. 读取 memory/YYYY-MM-DD.md
5. 读取 TODO.md ← 新增
6. 读取经验教训索引.md ← 新增
7. memory_search 搜索历史 ← 新增
8. 检查 HEARTBEAT.md ← 新增
9. 汇报进度
```

**从 4 步 → 9 步**，复杂度翻倍！

---

### 问题 2：局部最优 ≠ 全局最优

**每个机制单独看都对**：
- ✅ TODO.md 追踪任务
- ✅ 经验教训索引集中管理
- ✅ memory_search 避免遗忘
- ✅ Heartbeat 验证

**但合起来**：
- ❌ 文件太多（DL 难以查看）
- ❌ 流程太长（思考变慢）
- ❌ 维护成本高（每个都要更新）

---

### 问题 3：没有"机制审查"环节

**我们有**：
- ✅ 任务复盘（平江项目、发呆问题、Python 版本）
- ✅ 技术复盘（widemem 安装失败）
- ❌ **机制复盘**（新增规则是否合理）

## 全局优化方案（立即执行）

### 方案 1：文件合并

**合并前**（4 个文件）：
```
MEMORY.md（长期记忆）
memory/经验教训索引.md（单独索引）
TODO.md（任务追踪）
HEARTBEAT.md（定时任务）
```

**合并后**（3 个文件）：
```
MEMORY.md（统一记忆）
├── 核心协作规则（原经验教训索引）
├── 严重错误记录（所有复盘）
└── 核心目标

TODO.md（任务追踪）
HEARTBEAT.md（定时任务）
```

**收益**: 减少 25% 文件数量

---

### 方案 2：流程简化

**简化前**（9 步）：
```
1. 读取 SOUL.md
2. 读取 USER.md
3. 读取 MEMORY.md
4. 读取 memory/YYYY-MM-DD.md
5. 读取 TODO.md
6. 读取经验教训索引.md
7. memory_search 搜索历史
8. 检查 HEARTBEAT.md
9. 汇报进度
```

**简化后**（5 步）：
```
1. 读取 TODO.md（任务追踪）
2. 读取 MEMORY.md（含经验教训）
3. memory_search 搜索当前任务相关历史
4. 汇报进度
5. 等待 DL 指令
```

**移除**：
- ❌ 单独读取经验教训索引.md（合并到 MEMORY.md）
- ❌ 单独读取 HEARTBEAT.md（Heartbeat 时再读）
- ❌ 读取 SOUL.md/USER.md（已内化，不需要每次都读）

**收益**: 减少 44% 流程步骤

---

### 方案 3：建立"机制审查"习惯

**时间**: 每周日 21:00（深度 Heartbeat）

**审查清单**：
```
1. 本周新增了几个机制？
2. 哪些机制可以合并？
3. 哪些机制可以简化？
4. 整体流程是否过长？
5. DL 查看成本是否增加？
```

**目标**：
- 每周新增 ≤ 2 个机制
- 每月做 1 次减法（淘汰旧机制）
- 保持整体复杂度稳定

## 核心原则

> **简单 > 复杂**
> 
> **全局最优 > 局部最优**
> 
> **减法 > 加法**
